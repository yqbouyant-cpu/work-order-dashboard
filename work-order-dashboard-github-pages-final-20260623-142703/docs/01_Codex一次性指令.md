# 给 Codex 的一次性开发指令

请基于当前项目文件，开发一个“工单五模块管理看板”。

## 一、项目目标

我要做一个面向领导的工单管理看板。领导希望能在一个页面里查看所有工单是否及时结案、超期单数、具体未结案原因、当前卡点、下一步规划，并且可以按处理人员筛选，查看每个人名下的工单风险。

## 二、数据来源

本包内已有 4 类原始 CSV：

- sample_data/质量工单_260618 - 质量工单.csv
- sample_data/支持工单(资质文件)_260618 - 支持工单(资质文件).csv
- sample_data/市场工单_260618 - 市场工单.csv
- sample_data/供应工单_260618 - 供应工单.csv

## 三、五大模块

### 模块一：所有类型工单信息总汇总

必须展示：

- 工单总数
- 未结案总数
- 高风险超期单数
- 中风险单数
- 已结案数
- 涉及处理人员数量
- 按处理人员统计的工单数量、未结案数量、超期数量

模块一必须支持“下探”：

- 点击“下探/查看明细”后，可以看到具体工单列表
- 明细字段至少包括：工单类型、工单号、处理人员、制单时间、未结案天数、风险等级、单据状态、未结案原因、当前卡点、下一步规划、预计闭环时间、最新进展
- 未结案原因、当前卡点、下一步规划、预计闭环时间、最新进展需要设计为可编辑字段。若暂时没有后端，先用 localStorage 保存；后续预留飞书字段映射。

## 四、处理人员规则

处理人员根据制单人划分：

- 质量工单：处理人员 = 制单人
- 市场工单：处理人员 = 制单人
- 供应工单：处理人员 = 制单人
- 支持工单：如果没有制单人，则处理人员 = 创建人；如果创建人为空，则用 CC起草人

看板顶部必须有“处理人员”筛选器，领导可以选择某一个处理人员，页面所有统计和明细同步刷新。

## 五、风险规则

- 已结案：最终结案、工程师结案、已结案、闭环回访、关闭、结案
- 高风险：未结案天数 > 6
- 中风险：未结案天数 > 4 且 <= 6
- 低风险：未结案天数 <= 4

未结案天数 = 数据日期 - 制单时间日期。数据日期可以先取所有 CSV 中最大制单时间的日期；后续要做成可配置。

## 六、项目客诉专项模块

除了五大模块外，页面底部需要预留“项目客诉专项跟进”区域，用于项目类客诉管理。字段包括：

- 项目名称
- 客户/区域
- 关联工单号
- 处理人员
- 当前阶段
- 项目进度
- 寄回状态
- 寄回时间
- 分析结论
- 定责结论
- 现场处理方案
- 当前卡点
- 下一步动作
- 预计完成时间
- 最新进展
- 更新时间

当前阶段与进度映射：

- 受理建档：10%
- 现场排查：25%
- 寄回分析：45%
- 分析定责：60%
- 方案确认：75%
- 现场处理：90%
- 闭环回访：100%

页面上项目客诉最好用进度条展示。

顶部筛选器：

- 处理人员
- 工单类型

- 单据状态
- 搜索框：支持工单号、客户名称、问题内容、卡点搜索

- 下探明细可以展开/收起
- 手工填写字段可以保存到 localStorage；后续预留飞书多维表格 API 写回

## 八、视觉要求

请参考 prototype/工单五模块看板原型.html 的风格，但需要优化成更正式的企业管理看板：

- 主色蓝色，风险使用红/橙/绿
- 领导可读性优先，少堆文字
- 卡片式布局
- 明细表格要支持横向/纵向滚动
- 适配 16:9 大屏和普通电脑浏览器
