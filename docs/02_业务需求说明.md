# 业务需求说明：工单五模块管理看板

## 背景

现有工单已扩展为四类：质量工单、支持工单、市场工单、供应工单。领导希望通过看板查看：

1. 所有类型工单总体情况；
2. 哪些工单未及时结案；
3. 未结案原因是什么；
4. 当前卡点在哪里；
5. 下一步规划和预计闭环时间；
6. 可以按处理人员筛选，查看某个人名下的工单风险。

## 五大模块

### 模块一：所有类型工单信息总汇总

这是领导入口模块，需要支持下探。不能只看数字，要能点进去看具体工单、进度、卡点。

### 模块二：质量工单

关注客诉处理进度、售后上门状态、方案回复、结案情况。

### 模块三：支持工单

关注资质文件/支持诉求是否处理、审核状态、处理人跟进情况。

### 模块四：市场工单

关注市场/经销商服务类问题处理进度、分公司处理、经理审核、工程师结案。

### 模块五：供应工单

关注供应类问题方案回复、最终结案、补发/异常处理进度。

## 项目客诉专项跟进

项目客诉不适合只按普通工单统计，因为周期长，涉及寄回分析、分析定责、现场处理。需要单独模块呈现进度条和当前阶段。
