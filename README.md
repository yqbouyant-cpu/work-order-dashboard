# 工单五模块管理看板

这是基于 `prototype/工单五模块看板原型.html` 优化后的正式前端看板版本。

## 文件结构

- `index.html`：看板入口页面
- `src/styles.css`：页面样式
- `src/app.js`：数据读取、筛选、统计、下探和本地保存逻辑
- `data/工单统一明细_260618.csv`：当前前端使用的统一工单明细
- `data/项目客诉专项跟进模板.csv`：项目客诉专项跟进数据
- `data/embedded-data.js`：由 CSV 生成的本地兜底数据，用于直接打开 HTML 时展示
- `sample_data/`：4 个原始 CSV，保留用于字段溯源
- `normalized_data/`：统一后的明细和数据概览

## 本地运行

推荐用本地静态服务打开，这样页面会优先读取 `data/` 下的 CSV。

在本目录执行：

```powershell
python -m http.server 5173
```

然后访问：

```text
http://localhost:5173/
```

也可以直接打开 `index.html`。直接打开时，浏览器可能不允许读取 CSV，页面会自动使用 `data/embedded-data.js` 中的兜底数据。

## 当前功能

- 顶部业务 Tab：单日总汇总表、质量工单、供应工单、市场工单、支持工单、项目专项跟进
- 人员筛选：每个 Tab 按当前模块的制单人/创建人自动生成选项
- 数据范围：仅排除单据状态为“已结束”的工单，工单状态只作为流程节点展示和统计
- 单日总汇总表：总工单数、四类工单数量、高风险、中风险、有卡点工单
- 单日总汇总表新增：人员风险汇总、人员高风险工单数排名、TOP长周期未完结工单、工单明细看板
- 下探明细：高风险、中风险、有卡点、人员风险汇总、TOP长周期未完结工单、明细行均可打开明细抽屉
- 分类模块：工单总数、制单、已派工、已接单/已结单、已预约、服务中、暂不需上门
- 分类模块新增：人员高风险工单数排名、高风险工单清单、工单明细看板
- 风险规则：已流转天数超 10 天为高风险，超 6 天为中风险，4 天内为低风险，5-6 天为关注/待观察
- 项目客诉专项：按受理建档、现场排查、寄回分析、分析定责、方案确认、现场处理、闭环回访展示阶段进度
- 人工维护字段：未结案原因、当前卡点、下一步规划、预计闭环时间、最新进展、有卡点标记、风险原因、备注使用 localStorage 保存

## 后续接入飞书多维表格

当前版本的数据读取集中在 `src/app.js` 顶部的 `loadRows()` / `fetchText()` 流程。后续接飞书时，建议替换为：

1. 前端请求公司内部后端接口；
2. 后端读取飞书多维表格 API；
3. 后端把飞书记录转换为当前统一字段结构后返回前端；
4. 人工维护字段通过后端写回飞书多维表格。

不要把飞书 token、app secret 或任何密钥写入前端代码。前端只调用后端代理接口，密钥由后端或运行环境安全管理。
